# 命令参考

## /sdd-init
初始化需求，生成 Product Spec 草稿。

## /sdd-brainstorm
澄清需求，发散方案，识别风险和未知项。

## /sdd-design
根据锁定 Spec 生成 Design Doc。

## /sdd-plan
把设计拆成可执行任务。

## /sdd-breakpoint
保存当前进度和上下文。

## /sdd-recover
从断点恢复任务。

## /sdd-review
触发对 Spec / Design / Plan 的评审。

## /sdd-exec
按计划实施开发。

## /sdd-test
编写或执行测试，优先覆盖核心逻辑。

## /sdd-validate
检查代码、测试、文档是否与 Spec 一致。

## /sdd-archive
归档整个需求闭环。

## /sdd-help
显示当前可用命令与说明。
