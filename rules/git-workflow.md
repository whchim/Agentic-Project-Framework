# Git 工作流规范

## 1. 分支模型
- `main`：稳定主分支
- `develop`：集成分支（可选）
- `feature/*`：新功能
- `fix/*`：缺陷修复
- `hotfix/*`：紧急修复
- `release/*`：发布准备

## 2. 提交规范
建议使用 Conventional Commits：
- feat: 新功能
- fix: 修复
- docs: 文档
- test: 测试
- refactor: 重构
- chore: 杂项

## 3. PR 要求
PR 必须包含：
- 关联 Spec / Design / Plan ID
- 变更摘要
- 测试结果
- 风险说明
- 回滚思路（如适用）

## 4. 大任务隔离
- 大功能建议单独分支
- 高风险任务建议 worktree 隔离
- 不要把多个高风险任务塞进同一个工作区

## 5. 合并规则
- 测试通过
- 审查通过
- 文档同步
- 变更可追溯

## 6. 禁止事项
- 未说明就直接改主分支
- 未测试就合并
- 未写说明的大范围重构
